# Zh.A. & You — Daily Moments 💕

Персональный календарь-дневник для пары с планами, фото и воспоминаниями.

## ✨ Особенности

### 🏠 Главная страница
- Текущая дата и настроение дня
- Сегодняшние планы
- Ближайшие важные события
- Быстрый доступ к ключевым функциям

### 📅 Календарь
- Три режима просмотра: месяц, неделя, день
- Цветовая визуализация периодов отношений
- Отметки дней с записями и планами
- Стиль iPhone календаря

### 🕒 Лента моментов
- Instagram Stories-style фид
- Группировка по времени
- Поиск по моментам
- Полноэкранный просмотр фото

### 📌 Планы и напоминания
- Различные типы планов (прогулка, кафе, кино и др.)
- Повторяющиеся события
- Напоминания
- Отметка выполнения

### 💝 Важные даты
- Категории событий
- Годовщины
- Обратный отсчет
- Автоматическое отображение на главной

### 💌 Письма
- Короткие послания друг другу
- Уведомления о непрочитанных
- Полноэкранный просмотр

### 📊 Статистика
- Общая статистика
- Активные периоды
- Анализ данных

### ⚙️ Настройки
- Управление периодами отношений
- Отслеживание привычек
- Управление аккаунтом

## 🎨 Дизайн

- **Цветовая схема**: Пастельные оттенки розового, лавандового и голубого
- **Стиль**: Стеклянные эффекты, мягкие тени, закругления 18-24px
- **Анимации**: Framer Motion с fade и spring эффектами
- **Адаптивность**: Полная поддержка мобильных и десктопных устройств

## 🚀 Технологии

### Backend
- **NestJS** - прогрессивный Node.js фреймворк
- **Prisma** - современный ORM
- **SQLite** - база данных
- **JWT** - аутентификация
- **Multer** - загрузка файлов

### Frontend
- **Next.js 14** - React фреймворк с App Router
- **TypeScript** - типизированный JavaScript
- **Tailwind CSS** - utility-first CSS
- **Framer Motion** - анимации
- **Zustand** - управление состоянием
- **Axios** - HTTP клиент
- **Radix UI** - UI компоненты
- **date-fns** - работа с датами

## 📦 Установка

### Требования
- Node.js 18+
- npm или yarn

### Backend

```bash
cd backend
npm install
npx prisma generate
npx prisma migrate dev
npm run start:dev
```

Backend запустится на `http://localhost:3001`

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend запустится на `http://localhost:3000`

## 🔧 Конфигурация

### Backend (.env)
```env
DATABASE_URL="file:./dev.db"
JWT_SECRET="your-secret-key-change-this"
JWT_EXPIRES_IN="7d"
```

### Frontend (.env.local)
```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

## 📝 Первый запуск

1. Запустите backend
2. Запустите frontend
3. Откройте `http://localhost:3000/login`
4. Зарегистрируйте первого пользователя
5. Зарегистрируйте второго пользователя
6. Начните добавлять моменты!

## 🎯 Основные функции

### Добавление момента
1. Перейдите в Календарь
2. Выберите дату
3. Добавьте заголовок, описание, фото
4. Выберите настроение и место

### Создание плана
1. Перейдите в Планы
2. Нажмите "Добавить план"
3. Заполните детали
4. Настройте напоминание и повтор

### Управление периодами
1. Перейдите в Настройки
2. Добавьте период
3. Выберите даты и цвет
4. Периоды отобразятся на календаре

Цвета можно настроить в `frontend/app/globals.css`:
- `--primary` - основной цвет
- `--secondary` - вторичный цвет
- `--accent` - акцентный цвет

## 🐛 Известные ограничения

- Поддержка только двух пользователей
- Фото хранятся локально на сервере
- Нет push-уведомлений

## 🚀 Деплой

Подробная инструкция по деплою на VPS сервер находится в файле [DEPLOY.md](./DEPLOY.md)

### Быстрый старт для продакшена:
1. Клонируйте репозиторий
2. Создайте `.env` файл на основе `.env.example`
3. Скопируйте `google-oauth-credentials.json` в `backend/`
4. Запустите: `docker-compose up -d --build`
